# 3D Arena Survival Development Plan

## 1. 문서 목적

이 문서는 `PRD.md`를 실제 Unity 개발 순서로 풀어쓴 작업 가이드다.
목표는 "처음 3D 게임을 만드는 사람도 한 단계씩 따라갈 수 있는 구현 순서"를 만드는 것이다.

## 2. 개발 원칙

- 먼저 보이는 것부터 만든다.
- 복잡한 UI보다 게임 코어를 먼저 완성한다.
- 한 단계마다 "플레이해서 확인 가능한 상태"를 만든다.
- 모든 기능은 나중에 바꾸기 쉽도록 책임을 분리한다.

## 3. 권장 개발 단계

### 1단계. 씬 뼈대 만들기

목표:
- 게임이 올라갈 기본 무대를 만든다.
- 씬 구조를 이후 작업하기 쉬운 형태로 정리한다.

해야 할 일:
- `GameRoot` 루트 오브젝트 만들기
- `Managers`, `Arena`, `Players`, `UI` 자식 오브젝트 만들기
- 관전용 카메라 각도 조정
- 기본 바닥 배치
- 기본 매니저 스크립트 자리 만들기

완료 기준:
- 메인 씬을 열었을 때 구조가 한눈에 보인다.
- 이후 맵 생성과 플레이어 생성을 붙일 위치가 명확하다.

### 2단계. 경기장 생성

목표:
- 사각형 타일 맵을 코드로 만들 수 있게 한다.

해야 할 일:
- `ArenaManager`에서 맵 크기 값을 관리한다.
- 타일 프리팹 또는 기본 큐브로 맵을 생성한다.
- 타일 좌표를 코드에서 추적할 수 있도록 구조를 만든다.

완료 기준:
- 플레이하면 10x10 또는 20x20 경기장이 자동 생성된다.

### 3단계. 플레이어 프리팹 1종 만들기

목표:
- 움직일 수 있는 기본 캐릭터 한 종류를 만든다.

해야 할 일:
- `Capsule` 또는 `Cube` 기반 캐릭터 프리팹 생성
- `Rigidbody`, `Collider` 설정
- `PlayerController` 스크립트 연결
- 기본 이름과 색상 구분 준비

완료 기준:
- 캐릭터 하나를 씬에 놓고 물리적으로 정상 동작한다.

### 4단계. 플레이어 여러 명 생성

목표:
- 경기 시작 시 여러 명의 캐릭터를 자동으로 배치한다.

해야 할 일:
- `GameManager`에서 참가자 수 관리
- 스폰 위치를 경기장 위에서 랜덤 선택
- 기본 이름 자동 부여

완료 기준:
- 플레이 시 여러 캐릭터가 겹치지 않게 생성된다.

### 5단계. AI 자동 이동

목표:
- 플레이어가 직접 조작하지 않아도 캐릭터가 움직인다.

해야 할 일:
- 일정 시간마다 방향 갱신
- 경기장 밖으로 향하면 방향 수정
- 중심 복귀 성향과 랜덤성을 섞어 움직임 구성

완료 기준:
- 모든 캐릭터가 경기장 안을 자연스럽게 돌아다닌다.

### 6단계. 충돌 넉백

목표:
- 캐릭터끼리 부딪히면 전투처럼 보이게 만든다.

해야 할 일:
- 충돌 시 힘 적용
- 위로 살짝 뜨는 반응 추가
- 파워 값에 따라 넉백 강도 차등 적용

완료 기준:
- 부딪힘이 육안으로 읽히고 경기 흐름이 생긴다.

### 7단계. 탈락과 순위

목표:
- 맵 밖으로 떨어진 캐릭터를 탈락시키고 순위를 계산한다.

해야 할 일:
- 낙하 판정
- 탈락 순서 기록
- 마지막 생존자 우승 처리

완료 기준:
- 한 판이 자동으로 끝나고 1등이 결정된다.

### 8단계. 경기장 축소

목표:
- 시간이 지날수록 생존 공간을 줄여 긴장감을 만든다.

해야 할 일:
- 외곽 타일부터 제거
- 일정 주기마다 축소
- 위험 구역 연출 추가

완료 기준:
- 후반부로 갈수록 생존 공간이 눈에 띄게 좁아진다.

### 9단계. UI 연결

목표:
- PRD에 있는 사용자 흐름을 완성한다.

해야 할 일:
- 참가자 수 설정
- 이름 입력 필드 생성
- 시작 버튼과 카운트다운
- 결과 패널과 리셋 버튼

완료 기준:
- 사용자가 설정부터 결과 확인까지 한 번에 진행할 수 있다.

## 4. 초보자용 추천 작업 순서

처음에는 아래 순서만 집중하면 된다.

1. 씬 뼈대
2. 맵 생성
3. 플레이어 생성
4. AI 이동
5. 충돌
6. 탈락

여기까지 되면 이미 "재미 테스트 가능한 프로토타입"이다.
이후에 맵 축소와 UI를 붙이면 PRD의 핵심 흐름이 완성된다.

## 5. 현재 시작 지점

현재 프로젝트는 초기 씬만 있는 상태이므로, 1단계부터 차근차근 쌓아 올리는 방식이 가장 안전하다.
이번 작업에서는 1단계인 씬 뼈대 만들기부터 시작한다.

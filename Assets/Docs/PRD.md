# Product Requirements Document

## 1. 문서 정보

- 프로젝트명: 3D Arena Survival
- 문서 목적: 현재 구현을 기준으로 제품 요구사항과 다음 확장 방향을 제품 문서 형식으로 정리
- 기준 버전: 2026-03-18 코드 상태

## 2. 제품 요약

본 제품은 여러 AI 캐릭터가 경기장 안에서 자동으로 이동하고 충돌하며, 점차 좁아지는 안전 구역 속에서 최후의 생존자를 가리는 관전형 3D 서바이벌 게임이다.
사용자는 경기 시작 전 참가자 수와 이름을 설정하고, 게임이 자동으로 전개되는 과정을 관전한 뒤 결과 화면에서 우승자와 전체 순위를 확인한다.

## 4. 목표

--

### 제품 목표

- 누구나 10초 내 규칙을 이해할 수 있는 관전형 서바이벌 게임을 만든다.
- 짧은 한 판 안에서 긴장감이 상승하는 구조를 만든다.
- 참가자 이름 설정을 통해 가벼운 소셜/이벤트 활용이 가능하게 한다.

### 성공 기준

- 사용자가 시작부터 결과까지 단절 없이 한 판을 완료할 수 있다.
- 결과 화면에서 우승자와 순위를 즉시 이해할 수 있다.
- 참가자 수와 경기장 축소 속도 조정으로 템포 튜닝이 가능하다.

## 5. 비목표

- 플레이어 직접 조작형 액션 게임
- 온라인 멀티플레이
- 영구 성장/메타 프로그레션
- 아이템 파밍/인벤토리 시스템

## 6. 타깃 사용자

### 1차 타깃

- 짧고 단순한 관전형 미니게임을 원하는 캐주얼 사용자
- 스트리밍, 이벤트, 사내/학교 발표에서 이름을 넣고 결과를 보여주려는 사용자

### 2차 타깃

- 밸런스 실험과 자동 전투 로직을 빠르게 확인하려는 개발자/기획자

## 7. 핵심 사용자 시나리오

### 시나리오 A: 빠른 한 판 진행

1. 사용자가 씬을 연다.
2. 참가자 수를 정한다.
3. 이름을 입력한다.
4. 시작 버튼을 누른다.
5. 카운트다운 후 경기를 관전한다.
6. 최종 우승자와 순위를 확인한다.

### 시나리오 B: 반복 관전

1. 결과 화면을 본다.
2. 리셋한다.
3. 인원 수 또는 이름을 바꾼다.
4. 다시 한 판을 시작한다.

## 8. 기능 요구사항

### Must Have

- 참가자 수 조절
- 참가자 이름 입력
- 카운트다운 시작
- AI 자동 이동
- 충돌 넉백
- 축소 경기장
- 경기장 이탈 탈락
- 승자/순위 결과 UI
- 리셋

### Should Have

- 시각적으로 읽기 쉬운 캐릭터 스프라이트
- 경기장/위험 링 스프라이트 분리
- 문서화된 테스트 시나리오

### Could Have

- AI 성격 차별화
- 외곽 피해 누적
- 라운드 연속 진행
- 캐릭터 테마 스킨

## 9. 사용자 경험 요구사항

- 설정 UI는 최소 조작으로 시작 가능해야 한다.
- 플레이어 이름과 게이지는 화면에서 읽을 수 있어야 한다.
- 경기장 축소는 시각적으로 명확해야 한다.
- 결과 화면은 우승자 강조가 필요하다.

## 10. 기능 상세

## 10-1. 맵 기능
    -- 맵은 사각형 타일의 집합으로 구성된다. (ex: 10x10 or 20x20)
    -- 맵 타일은 시간이 지남에 따라 외곽라인부터 부셔짐 (맵 축소)
    -- 랜덤으로 맵 중간의 1개의 타일이 부셔짐 

## 10-2. 캐릭터 기능
    -- 캐릭터는 1x1x1 정도의 귀여운 캐릭터 (닭, 펭귄, 기사, 슬라임 등)
    -- 캐릭터는 ai를 통해 스스로 움직임 (타일이 없는 곳으로 스스로 나가면 안됨, 캐릭터간 충돌 시 위로 살짝 튀어오르며 넉백, 각 캐릭터는 파워게이지(1~10 시간별 증감))

### F-01 설정 단계

- 인원 범위는 4~12명
- 이름 입력 필드는 인원 수에 맞춰 생성
- 빈 값은 기본 이름으로 대체
- 키보드 Tab 이동 지원

### F-02 경기 시작 단계

- 이전 경기 상태 정리
- 플레이어 생성
- 카운트다운 진행
- 카운트다운 종료 시 물리 활성화

### F-03 전투 단계

- 플레이어는 AI 로직으로 자동 이동
- 중심 복귀 성향과 타깃 추적 성향 포함
- 충돌 시 넉백 적용
- 파워 게이지가 넉백 강도에 영향

### F-04 후반 압박 단계

- 일정 시간 후 경기장 축소 시작
- 위험 링 색상 및 펄스 연출 강화

### F-05 종료 단계

- 마지막 생존자 1등 판정
- 탈락 순서 기준 나머지 순위 부여
- 결과 패널에 우승자와 랭킹 표시

## 11. 비기능 요구사항

- 인스펙터 기반 파라미터 조정 가능성
- 최대 12명 기준에서 자연스러운 진행
- 구조상 GameManager / ArenaManager / PlayerController 책임 분리
